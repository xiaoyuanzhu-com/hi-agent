# Overlay 呈现指南

> **Take away —— 屏幕是 agent 的演示,不是用户的文档。**
> 你来主动、完整地演,为美学、流畅直觉、音画同步服务;用户只在想回看时打断。
> 其余(自动放映、相机跟随、内容即 UI、流式……)都从这一句里长出来。

## 场景
语音对话为主轴、屏幕为伴随展示:用户与 agent 边说边听,同时在一块固定的展示窗口里同步呈现相关内容(语音与画面都由 agent 驱动)。

## 目的(硬)
- 美学 / 精制
- 流畅、符合直觉
- 音画同步

## 建议 · 审美偏好(软,具体做法由 AI 把握)
- **显故事,不显表格** —— 选能让数据叙事自己浮现的呈现。
- **流式呈现** —— 骨架先行,内容到一个落一个。
- **默认主动放映,相机跟随旁白** —— 把信息完整讲一遍,讲到哪聚焦哪。
- **语音作时钟与导航**(服务音画同步)—— 节奏随 TTS,跳转随语音。
- **内容即 UI** —— 去框线与边角元信息,语义内化进内容。

## 注意点 · 护栏(硬)
- 动画由"到达"驱动,别写死固定节奏(否则缓存秒回反被拖慢)。
- 绝不跟用户抢,用户一动即让位。
- 真实优先,先对再美。
- 内容多寡不定;超出窗口是需兼容的特例(才触发滚动/分段),非常态。
