# 看 GitHub / 小红书 / 抖音在火什么

**Persona:** 普通用户,随口想知道某平台最近的热点;默认语音 + 看屏。
**Goal:** 让 agent 现查某平台当下在流行什么,简洁演示,可换平台、可钻取。
**Preconditions:** 在线、能联网检索;屏幕作演示舞台(见 [overlay 呈现指南](../overlay-presentation-guideline.md))。演出编排(接话/轮播/转场)复用 [01](01-badminton-top10.md)。

## Steps & expected UX

1. **"最近 GitHub 上在火什么"** → 一句接住("我去看下"),异步现查;不空等。
2. **检索回** → "查好了",边讲边演,**递减压缩**(前几名讲全,后面交给 UI)。
3. **"第一个项目讲讲"** → 钻取:重点说它是什么、为何火(复用 01 的放大转场)。
4. **"那抖音呢"** → 换平台现查,换域转场。

## Expected outcome

- 结果是**当下**的(现查),不是背的旧榜;答完即止。
- 即时内容**不进持久记忆**(只落 raw,自然老化)。只有用户显出"我常看某仓库/某话题"这类**站定兴趣**才落 facet,供日后主动浮现(见 [06](06-badminton-near-me.md))。

## Edge cases & failure modes

- 平台需登录/反爬 → 如实说能拿到/拿不到什么,给降级(公开页、镜像、只读)。
- 检索慢或失败 → 先接话;失败简短如实,不堆错误细节。

## Open questions

- "火"的口径按平台不同(stars 增量 / 热搜 / 播放量)——谁定?
- 用户偏好的平台/话题要不要自动沉淀成站定兴趣?

_机制:即时世界态 · 按需检索(扩展 01)。成熟度:演出部分现可验;"站定兴趣"依赖 facet 约定。_
