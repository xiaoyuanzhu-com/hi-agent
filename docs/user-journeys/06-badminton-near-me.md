# 我附近最近有什么羽毛球活动

**Persona:** 羽毛球迷,住在某城;希望 agent 知道"我爱羽毛球、我在哪",并据此关照我。
**Goal:** 现查赛事/活动,用我的兴趣 + 位置过滤;值得时主动提一句"本周附近有…"。
**Preconditions:** agent 已知"用户是羽毛球迷""所在城市"(facet);在线、能联网;pulse 在跑。**这是 [01](01-badminton-top10.md) 的延伸**——01 立了即时检索的演出,06 加上**个性化 + 主动浮现**。

## Steps & expected UX

1. **"最近有什么羽毛球活动?我附近呢"** → 现查 + 用站定兴趣(羽毛球)+ 位置过滤,演示**与我相关**的活动,而不是泛泛全国榜。
2. **(异步)** pulse 里发现"本周末附近有一场" → 空闲时**主动顺嘴提一句**;不打断、可叫停("先别管这个")。
3. **位置/兴趣更新** "我搬到上海了" → facet 更新,之后的过滤随之变。

## Expected outcome

- 结果明显**带个人色彩**(我的项目、我的城),不是通用热搜。
- 主动那一句出现在**恰当的空闲时机**,且**可被叫停**;叫停后不再唠叨。

## Edge cases & failure modes

- 不知道位置 → 先问一次或用已知 facet,不臆测;拿到后记住。
- 附近确实没活动 → 老实说"附近这周没看到",不硬凑。

## Open questions

- "附近"的位置从哪来:用户给定的 facet,还是设备定位?粒度(城/区)?
- 主动浮现的社交时机:怎样算"恰当空闲"?频率上限?
