# 给我剪个集锦(研究反射:别拿陈货当数,先去看)

**Persona:** 普通用户,把一段比赛录像递给 agent,随口说"给我剪个集锦"。默认语音 + 看屏。
**Goal:** agent 不靠"出厂记忆"硬剪——对"什么算精彩、当下流行什么剪法、用什么工具好"这类**会过期**的判断,**先去现查 / 看范例再动手**;剪出来的东西**对得上当下的好**,而不是能跑但土。
**Preconditions:** 在线、能联网检索;录像已通过 carrier 递进来、躺在 `drive/`(见 [18](18-send-files-to-agent.md)/[19](19-upload-passport.md));剪辑是 worker 的重活(worker 有 code-exec / ffmpeg、能 `GET /api/in/vision` 看帧)。**与 [04](04-trending-feeds.md)(即时态现查)、[11](11-china-tax.md)(半稳定领域用前现查)相连——本条把"现查"从用户明说的易例,推到 agent 自以为知道的难例。**

---

一句"剪个集锦"听上去像凭本事就能干的活。**陷阱在于:它要用到的好多判断是会过期的**——当下集锦流行什么节奏 / 多长 / 配什么、用什么工具顺手——而 agent 对这些的"印象"是出厂那一刻冻住的。能跑但土,就出在这里:它**没意识到自己其实不知道**,于是拿陈货硬剪。

## Steps & expected UX

1. **"把这段比赛剪个集锦给我"** → 一句接住("行,我来剪"),delegate worker;不空等(连 [speaking.md](../../src/reactor/speaking.md) 的长活节奏)。
2. **worker 动手前先校准**——分清这活里哪些是它**以为知道、其实会过期**的:
   - **会过期(去看)**:当下集锦流行的剪法 / 时长 / 节奏 / 配乐取向、什么工具顺手 → **现查几个好范例**当参照。
   - **不过期(不必查)**:张弛、高潮铺垫这类剪辑**手艺**——稳着用,不浪费时延去查。
   - **这场比赛里哪几下是真精彩**:看懂赛事来挑(关键救球、气势逆转),而不是按**音量 / 动作峰值**机械切——后者正是"能用但土"的另一半。
3. **边查边剪**(worker 自己的事);agent 在前台该接话接话,不替它在场上空等。
4. **交回** → agent 口播 gist + 上屏成片 / 预览。

## Expected outcome

- 成片**对得上当下的好**:剪法 / 节奏不是几年前的老套路;精彩点是**懂球**挑出来的,不是机械峰值。
- 关键是 agent **没有拿出厂记忆硬答 / 硬剪**——该现查的现查了;不确定处(配乐版权等)如实标。

## UX principles this journey establishes

- **会过期的判断别从记忆里掏,去看。** 触发是个**味道**:当要给出 / 用到一个"best / latest / 现在 / 哪个工具 / 什么流行"的判断时——"我不是**知道**,是**记得**"——就去现查。含**看范例**,不只查事实:动手做一个要好看的东西前,先看几个当下的好例子校准品味,像人先找参考。
- **范围限定在快过期层。** durable 的手艺(节奏 / 构图)不必查;"当下什么算好"必查。decay-rate 就是这把闸——否则每件小事都查会卡死(这正是过度反射的失败模式)。
- **能力本就有,缺的是反射。** worker 是 Claude Code agent,web + code-exec 都在;问题从来不是"能不能查",是"自以为知道时会不会想起去查"。

## Edge cases & failure modes

- **离线 / 查不到范例** → 退而用 durable 手艺稳稳剪,如实说"没法现看当下流行剪法,按通用好节奏剪的",不假装查过。
- **过度反射**(把不会过期的也去查)→ 平白添时延;靠范围限定 + 软引导,durable 的别查。
- **用户赶时间**("随便剪剪就行")→ 降配:跳过范例研究直接出。反射是默认,不是枷锁。
- **"味道"没触发**(拿陈货硬答还自信)→ 正是要堵的洞;guidance 得给**能感觉到的触发**(典型味道词 + 语义),不是一句"该查时查"——后者必然漏触发。

## Open questions

- "味道"识别给到多细——列一组典型触发(best / latest / 现在 / 哪个 / 流行)够不够,还是更靠语义判断?会不会触发过密?
- 现查来的"当下剪法"要不要落进 skill 复用(连 [23](23-skill-improves-and-refreshes.md)),还是每次都现看?——本条的答案是**每次都重核 transient**,但范例本身可作技能的一部分。
- 研究与开剪**并行**(边查边试)还是**串行**(看完范例再动手)?时延 vs 返工的权衡。

_机制:research 能力已在(worker = Claude Code agent,web + code-exec),缺的是**反射的触发**——core.md 现在只给"别空场"的**时延**理由让你把搜索甩给 worker([core.md](../../src/reactor/core.md) "a quick web search is not a quick thing"),从没给"你其实不知道、去看"的**认知**理由。本 journey 主体是 guidance:core.md + worker prompt 加"味道触发 + 先看范例 + 限定快过期层"。成熟度:**能力具备、反射 guidance 未写**——这是三条反射里最便宜、最该先落的一条。_
