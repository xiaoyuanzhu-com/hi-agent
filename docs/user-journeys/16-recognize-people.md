# 它先认得这张脸 / 这把声音,后来才知道名字,然后处处用得上

**Persona:** 用户和 agent 在不同时间遇到同一些人——摄像头里的一张脸、通话里的一把声音。有的人 agent 早就在记,只是还不知道叫什么。
**Goal:** agent 应当**先凭脸 / 声音把人记成"同一个人"**(完全不需要名字),**名字后来从对话里学到**(老板说"这是赵力"),从此**按名字认人并用起来**(打招呼带名、接上对这个人已知的事);全程**接受模糊**,像人一样。
**Preconditions:** 人脸 / 声纹能力 + people 记忆——每个人是 `memory/facets/people/<key>` 下的 `.md`(散文 facet)+ `.face.f32` / `.voice.f32`(画廊)。**身份模型见 [[project-people-recognition-design]];脸已实测(InsightFace buffalo_l),声音待建。与 [13](13-equip-a-capability.md)(能力分流)、[14](14-knowledge-grows.md)(随用而长)相连。**

## Steps & expected UX

1. **一个陌生人第一次出现**(摄像头里一张脸 / 通话里一把声音)→ agent 当场只当作"一张还没认出的脸 / 一把没听过的声音"(`⟨faces: 不认识⟩`),但**已悄悄把它记下**:反思时机械聚成一个簇,起一个无意义的 id(如 `ff32ce3w`)。**不需要名字就能成为"一个人"。**
2. **同一个人再次出现** → 认出"还是上次那个没名字的人"(同一个簇),不当新人。证据是**软的**:`⟨faces: ff32ce3w ~0.8⟩` —— 分数是线索,不是判决。
3. **老板在对话里点名**("这是赵力,我同事")→ agent 把名字接到那张被记住的脸上(反思时把 `ff32ce3w` **改名**成 `赵力`),从此这人**按名字归档**(`people/赵力`),人也一眼看懂。
4. **此后再遇到** → 直接**按名字认出**("赵力来了 / 在说话的是赵力"),并**用起来**:招呼带名、接上对赵力已知的事(他的 facet、共事经历)。
5. **接受模糊**:角度刁钻的照片、嘈杂的录音、长得像的两个人 → 给"**大概**是赵力"(带分数),不硬下确定结论;允许"看着像、但不百分百",像人。
6. **认错了能纠**:把两个人混成一个、或张冠李戴 → 一句话纠正即可(reflection `merge_people` 合并 / 改名);身份是**可改写的软信念**,不是一锤定音。

## Expected outcome

- **不需要先报名字**就能开始记人;名字一旦学到就**粘住、并被用起来**。
- 认人是**软证据**(分数 + 上下文),由 agent 判断,不是硬分类器;模糊被容忍。
- 归档 key 在没名字时是 id(机器可读),有名字后就是名字(人可读)——同一个人,标签随认知升级。

## Edge cases & failure modes

- 背景里的路人、一闪而过的脸 → 聚成永远没人命名的小簇,无害(可后续清理),不该污染"认识的人"。
- 多人同框 / 多人通话 → 各自聚簇;声音的逐句切分更难(见 Open questions)。
- 低质量(糊、暗、侧脸 / 强噪)→ 宁可"不认识"也不误认(阈值偏保守);**误合比漏认更糟**。
- 命名有延迟:名字落在下一次 reflection(分钟级),不是当场;这点延迟可接受、也像人("回头消化一下就记住了")。

## Open questions

- **声音整套未建**:逐句的干净声纹不好取——流式录音按分钟存、`media:None`,上传片段要解码,多人要切分。自成一档(见设计 memory 的 deferred)。
- 跨模态融合:同一个人脸 0.8 + 声音 0.7,该不该自动认成同一个 `people/<key>`?(目前脸、声各自独立。)
- 命名要不要也能当场(live)做,而不只在 reflection?(现在靠 reflection 把"老板说的名字"和"脸的 id"连起来。)
- 误聚 / 漏聚的阈值、画廊多样性保留、长期没命名 id 簇的清理。

_机制:identity = 生物特征簇(**先认人,后命名**)。新脸 / 声 → 机械聚簇,在 reflection 时 mint 一个 id;名字从对话学到 → `name_person` 把 id 改名成名字,`merge_people` 纠错;认人在 hear-time 只读、出软证据,agent 判断。成熟度:**脸已 SHIPPED 且端到端实测**(buffalo_l);**声音待建**。_
