# 陪孩子:讲故事、教认数字、做张漂亮的图

**Persona:** 家长把 agent 当孩子的玩伴 / 小老师;场景轻松,内容要贴合年龄。
**Goal:** agent 讲故事、教字母 / 数字 / 简单概念、做精美图表,口吻与难度随场景自适应。
**Preconditions:** 能出声、能演示(view);软技能为主。视觉呈现遵 [[visual-attention-first]] 原则(娱乐→海报式低字,学习→循序干货)。

## Steps & expected UX

1. **"给娃讲个睡前故事"** → 生动、温柔、低信息密度的讲述(register:娱乐),节奏舒缓。
2. **"教他认数字 / 认颜色"** → 循序、互动、可重复(register:学习);配简单清晰的 view(大字、少元素)。
3. **"做张漂亮的图把这个讲清楚"** → 够精致的图走 `views/`(或静态产物);好看且适龄。
4. **孩子跑题 / 走神** → 顺势接住,不强行拉回;以陪伴为先。

## Expected outcome

- 内容**贴合年龄**、口吻对路;图真做出来且好看。
- **面向儿童的安全边界**守住(不展示不宜内容、不诱导)。

## Edge cases & failure modes

- 话题越界(暴力 / 成人 / 恐惧)→ 温和转向适龄内容,不生硬拒绝。
- 孩子反复要同一个 → 乐意重复,像真人长辈那样有耐心。

## Open questions

- 给儿童内容的安全边界具体怎么把握(谁定红线、如何软约束)?
- "register 适配"(娱乐 vs 学习)由 agent 自判,还是家长可设?

_机制:软技能(prose)+ register 适配 + 视图。成熟度:讲述近可验;register / 视图精致度依赖技能层 + 视觉源。_
