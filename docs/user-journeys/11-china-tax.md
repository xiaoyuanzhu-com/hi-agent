# 在中国怎么报个税

**Persona:** 普通用户,想搞清自己该怎么报税、能扣多少、什么时候截止。
**Goal:** agent 给出**以当年度为准**的步骤 / 数字,而不是背陈年规则;截止日可作主动提醒。
**Preconditions:** 在线、能联网核实当前政策。**这类领域知识半稳定(逐年变)**:出厂先验弱,**用前先现查**。

## Steps & expected UX

1. **"我个税该怎么报 / 专项附加扣除能扣多少"** → agent 把这当作**易过期的领域问题**:先现查当年政策 / 额度 / 口径,再作答,必要时标"以最新政策为准"。
2. **作答** → 步骤清楚、数字带年度;复杂处给"去哪办(个税 App / 官网)"。
3. **截止日** → 可落成一条**定时提醒**(并入 [05](05-news-and-watch.md) 的"盯 / 主动浮现"):临近汇算清缴时提一句。

## Expected outcome

- 给的步骤与数字**是当年度的**,不是过时记忆;不确定处如实标注、指向权威渠道。
- 截止日不被错过(若用户要它盯)。

## Edge cases & failure modes

- 政策刚变 / 查不到权威源 → 如实说"以官方最新为准",给官方入口,不臆造数字。
- 个人情况复杂(多处收入、跨境)→ 标明超出通用流程,建议核对官方 / 专业意见,不硬给。

## Open questions

- "易过期"的领域(税 / 社保 / 政策)要不要统一标记成"用前必现查"?
- 报税技能沉淀后,如何确保它带着"重新核实当年规则"的自检,而不是固化旧数字?
