# 播放音乐(开 app → 搜放 → 投屏 → 记住偏好)

**Persona:** 用户随口一句"播放蜗牛与黄鹂鸟",想听歌;不关心用哪个 app、怎么搜怎么点。
**Goal:** agent 把歌**真的放出来**——有音乐 app 就用,没有先(请示后)装一个;搜到并播放;愿意的话把播放界面投成一个 view,用户随时可收起让它在后台放;**第二次更顺**(记住用哪个 app、怎么搜放、要不要投屏)。
**Preconditions:** 在能控到 GUI 的机器上(Mac mini 在场;无头服务器不行,见 [08](08-operate-apps.md))。装 app 属"置备",需请示(范型见 [02](02-feishu-sprint-backlog.md)/[07](07-browser-errand.md))。投屏 = 窗口采集→视频流进 view(net-new)。控曲:多数音乐 app 无脚本 API,走 vision+click(UI 自动化档)+ 媒体键。

## Steps & expected UX

1. **"播放蜗牛与黄鹂鸟"** → 接住;先看这台机器上有没有音乐 app(QQ 音乐 / 网易云音乐等)。
2. **有** → 直接开(`open -a`),在里面搜这首、播放。
3. **没有** → **装一个是任务的一部分**:提议装(如 QQ 音乐),**请示后**再装,然后继续。不擅自装(装 app 要先问,见 CLAUDE.md / [02](02-feishu-sprint-backlog.md))。
4. **放起来后** → 把播放界面投成一个 view(窗口采集成视频流),用户看得到正在放什么。
5. **"我去忙别的 / 把这个收了"** → 收起 view,**音乐继续在后台放**(原生 app 不受 view 关闭影响)。区分"收画面"(继续放)vs"别放了 / 停"(停播)。
6. **控制** → 暂停 / 下一首 / 音量,语音即可,不必盯着 view(媒体键 / UI 自动化)。

### 第二次:更顺

7. **再说"播放蜗牛与黄鹂鸟"** → 明显省事:
   - **跳过"扫有哪些 app"** —— 记住了上次用哪个。
   - **直接按上次学会的"在这个 app 里搜+播"操作** —— 怎么搜放沉淀成技能,随用而长(见 [14](14-knowledge-grows.md);[[no-prebundled-assets-accumulate-via-guidance]])。
   - **按上次的 view 偏好** —— 上次收起去忙了 → 这次默认后台 / 给个小卡片,不强行投全屏。少问一句。

## Expected outcome

- 歌**真的在响**(可验:app 在前台/后台播放该曲),不是"我建议你打开 QQ 音乐搜一下"。
- view 可投可收;**收了不停播**。
- 第二次明显更省事:不重复发现、不重复试错、按记住的偏好来。
- 碰不到 GUI / 装不了 app / 歌是 VIP → 如实说清边界 + 替代,不假装放了。

## Edge cases & failure modes

- 无头 / SSH 无 GUI → 控不了 GUI 音乐 app,诚实降级(见 [08](08-operate-apps.md));或退化为可命令行播放的源。
- app 没装且不让装 / 区域不可用 → 老实说,给替代(网页版 / 另一个 app)。
- 歌是 VIP / 会员专享 / 未登录 / 有广告 → 如实报边界,问要不要登录或换源,不硬闯付费墙([[careful-irreversible-actions]])。
- app 改版 / 搜索框控件变了 → 重试一次;vision+click 不稳则如实降级。
- 投屏需 Screen Recording 授权 → 一次性引导开通,说人话;没授权就**只播不投**。
- 同名歌 / 多个版本 → 取最匹配的播,一句话说选了哪个,可纠正。
- "关掉"歧义 → 默认理解为收 view 继续放;不确定就一句确认,别误停。

## Open questions

- 投屏到底投**原生窗口像素**,还是 agent 自画一个 now-playing 卡片?(overlay 模型:屏是 agent 的演示——倾向默认自画卡片,用户要"看真界面"才投像素。)
- "用哪个 app + 怎么搜放"落**技能**还是**记忆**?(knowledge-grows [14](14-knowledge-grows.md) 范型:认识/技能按需加载。)
- view 偏好的粒度:按任务类型(听歌默认后台)还是全局?记成 preference 一条。
- 收起 view 后控曲入口只有语音吗?要不要留个极小的常驻控件?

_机制:effector(open app + UI 自动化/vision-click + 媒体键)+ 窗口投屏(采集→流进 view)+ 技能(怎么在该 app 搜放)+ 记忆/偏好(用哪个 app、投屏与否)。可行性:**可行但逐 app**;多数音乐 app 无脚本 API,控曲走 vision+click 档(见 [08](08-operate-apps.md) 可行性排序)。成熟度:依赖 effector + 投屏 + 技能层(均未建)。_
