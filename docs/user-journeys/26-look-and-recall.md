# 看懂一帧:举起来当场问 / 存下来回头找

**Persona:** 用户把一张图给到 agent——举起实物对着摄像头,或随手发来一张照片。
**Goal:** agent 看懂这一帧的**真实内容**:既能当场答到点上,这份"看懂"又能留存,让图日后按**画面内容**被找回。
**Preconditions:** 有通用视觉能力(frame endpoint:image + prompt → 文字理解)。与 [16](16-recognize-people.md)(脸是另一路软证据、内置模型)、[19](19-upload-passport.md)(文件当物件、不走这条感官)区分开。

## Steps & expected UX

### 当场问(pulled · 答这一刻)
1. **举起一盒外文药 / 一张外文菜单**:"一次吃几粒?" / "这菜有没有花生?" → agent 抓一帧,带着**这个问题**去看 → 直接答**剂量 / 含不含花生**,不是泛泛"这是一盒药"。
2. **手机对着笔记本上的报错**:"这报错啥意思、怎么修?" → 跨设备读屏(screen-control 够不到另一台机器,这里用摄像头看)→ 解释 + 给修法。
3. **追问"那这个呢?"**(再举一样)→ 顺上下文看下一帧,不用重述。

### 回头找(on-ingest · 记住以备日后)
4. **发来一批照片**(海边、白板、一张收据)→ 落存的同时每张过一遍视觉,把"画面里有什么"记成文字(海滩日落两人 / 白板上三个框的架构图 / 某店收据 ¥128)——文字进记忆,原图进 annex。
5. **几天后"找那张海边的"** → 按**内容**召回,不靠文件名 / 时间。
6. **"上周二白板那张,整理成文字"** → 看懂白板 → 出结构化笔记。

## Expected outcome

- 答的、找回的都是画面里的**具体信息**(剂量、成分、报错原因、收据金额),不是一句"这是一张图"。
- 存图的字幕从 `[vision capture]` 这类占位升级成**真看懂**;召回靠画面内容即可成立。

## Edge cases & failure modes

- **看不清 / 反光 / 角度差** → "换个角度再拍一张",不瞎猜。
- **敏感件**(证件、病历)→ 默认私密、不主动外传;要外发先确认意图(连 [19](19-upload-passport.md))。
- **医疗 / 健康类判断**(看皮疹、看药)→ 给信息、不下诊断;严重的引导就医。
- **量大** → 只褪**原图字节**、文字理解永久保留(连 forgetting:媒体可褪、事实常驻)。

## Open questions

- "当场问"的 prompt = 用户那句话;"回头找"没人在场时,默认 prompt 该多细(只记显著物体?还是连场景氛围)——成本 vs 召回质量。
- 升级 caption 是对**所有**入库图都跑,还是有门控(贵)?

_机制:通用视觉 frame endpoint(image + prompt → 文字)+ 现有 caption 路径升级(`server/vision.rs` 那句固定 `"Describe what you see, briefly."`)。"看懂"落进记忆,与 everything-is-memory 同流。成熟度:capability seam 已在(`vision::understand`),endpoint 接入中,未实测。_
